How I Built a WireGuard Mesh Across 7 Nodes for $6/month

5 Raspberry Pis behind residential NAT, 2 cloud servers with public IPs, all connected via WireGuard. Full SSH from anywhere. Total VPN cost: one $6/month DigitalOcean droplet.

The Problem

I have 5 Raspberry Pis at home behind a consumer router with no static IP and no port forwarding. I also have 2 DigitalOcean droplets. I need all 7 nodes to talk to each other securely, and I need to SSH into any Pi from anywhere.

Tailscale solves this but costs money at scale and adds a dependency. I want to own the VPN too.

The Architecture

Hub-and-spoke with full mesh routing. Anastasia (DO droplet, $6/month) is the WireGuard hub. Every Pi connects outbound to Anastasia on port 51820. Since the Pis initiate the connection, NAT traversal works automatically.

Internet | Anastasia (174.138.44.45:51820) ── WireGuard Hub | | | | | | Alice Cecilia Octavia Aria Lucidia Gematria 10.8.0.6 10.8.0.3 10.8.0.4 10.8.0.7 10.8.0.5 10.8.0.8 (.49) (.96) (.101) (.98) (.38) (DO nyc3)

Every node gets a 10.8.0.x address on the WireGuard interface. The hub routes packets between all peers. From my laptop (also a WireGuard peer), I can SSH to any Pi by its WireGuard IP.

Setup

On the hub (Anastasia):

[Interface] Address = 10.8.0.1/24 ListenPort = 51820 PrivateKey = <hub-private-key> PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE [Peer] # Alice PublicKey = <alice-public-key> AllowedIPs = 10.8.0.6/32

On each Pi:

[Interface] Address = 10.8.0.6/24 PrivateKey = <alice-private-key> [Peer] # Hub PublicKey = <hub-public-key> Endpoint = 174.138.44.45:51820 AllowedIPs = 10.8.0.0/24 PersistentKeepalive = 25

The PersistentKeepalive = 25 is critical. Without it, the NAT mapping expires and the hub can't reach the Pi. With it, each Pi sends a keepalive packet every 25 seconds, keeping the NAT hole open.

What It Enables

  • SSH from anywhere — ssh blackroad@10.8.0.6 reaches Alice regardless of my location
  • Service routing — Gitea on Octavia (10.8.0.4:3100), Ollama on Cecilia (10.8.0.3:11434), all reachable across the mesh
  • TLS edge — Gematria (10.8.0.8) runs Caddy, terminates TLS for 18 domains, and forwards to Pis via WireGuard
  • Zero port forwarding — no router config needed, no static IP needed

The TLS Edge Pattern

Internet traffic hits Gematria (public IP, Caddy with Let's Encrypt). Caddy reverse-proxies to the Pis via WireGuard IPs:

search.blackroad.io { reverse_proxy 10.8.0.6:8094 # Alice via WireGuard } chat.blackroad.io { reverse_proxy 10.8.0.3:11434 # Cecilia Ollama via WireGuard }

From the outside, it looks like a normal HTTPS website. Internally, every request traverses an encrypted WireGuard tunnel to a Pi sitting in my living room.

Cost

  • Anastasia (hub): $6/month DO droplet (1 vCPU, 1GB RAM)
  • Gematria (TLS edge): $12/month DO droplet (4 vCPU, 8GB RAM — also runs Ollama)
  • WireGuard: free (kernel module on all nodes)
  • Pis: already owned, ~40W total power

Total VPN cost: $6/month. The $12 Gematria droplet is shared with Caddy, Ollama, and NATS — it's not a VPN cost.

Compared to Tailscale

Tailscale is easier to set up. But: (1) it's a dependency I don't control, (2) the free tier limits devices, (3) I can't customize routing, and (4) I actually run Headscale (self-hosted Tailscale control plane) on Aria as a backup. The WireGuard mesh is the primary.

For a homelab with 7 nodes, raw WireGuard is simple enough. The config files are 15 lines each. Once set up, it just works.
