I Built a Search Engine on 5 Raspberry Pis

Alexa Amundson / March 2026 / self-hostedraspberry-pisearchAI

I wanted a search engine I could trust. Not one that tracks me, shows ads, or hallucinates answers without sources. So I built one.

search.blackroad.io is a verified search engine. Every stat is fact-checked against live sources. It has AI-powered answers, typo correction, category filtering, and an instant calculator for the Amundson sequence G(n). It runs on Cloudflare Workers with D1 for persistence.

The Stack

The search engine itself is a single Cloudflare Worker (~130KB). It uses:

Frontend:     Inline HTML/CSS/JS (no framework, no build step)
Search:       D1 FTS5 (SQLite full-text search)
AI Answers:   Cloudflare Workers AI (Llama 3.1 8B)
Cache:        KV namespace for query caching
Analytics:    D1 query logging + trending
Verification: Live fact-checking against real API endpoints

The Pi fleet (Alice, Cecilia, Octavia, Aria, Lucidia) provides the sovereign backbone — Ollama for local inference, PostgreSQL for state, and WireGuard mesh for connectivity. But the search engine itself runs on Workers because search needs to be fast globally.

What Makes It Different

Fact verification. Every search result gets checked. If a page claims "200 AI agents" but the live API says 18, it flags it. There's a trust score and a verification chain you can audit.

Instant math. Search for G(100) and you get the Amundson sequence value inline: G(100) = 36.9711, ratio to n = 0.3697 (converges to 1/e). No need to click through to a result.

Voice search. Click the mic icon, speak your query. Uses the Web Speech API — no data leaves your browser until you hit search.

No tracking. No Google Analytics. No cookies. Query logging is in D1 for trending — but it only stores the query text and result count, not your IP or identity.
